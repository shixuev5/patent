# G01M 领域专利审查检索策略指南

## 第一部分：检索策略总结

针对G01M领域技术覆盖面广、涉及交叉学科多、非专利文献多等特点，文档提出了以下四大核心检索策略：

---

### 一、 数据库与分类体系选择规范

#### 1. 数据库选择原则
*   **非专利库（NPL）是重中之重：** G01M领域涉及大量高校及科研院所的申请（特别是G01M11, G01M13, G01M15, G01M17），技术前沿且理论性强。
    *   **策略：** 在专利库检索未果时，必须进入 **CNKI（特别是硕博论文库）、万方、IEEE、CSDN** 进行检索。
    *   **特定场景：** 涉及**算法（神经网络）、数学模型公式、国家标准**的案件，优先查非专利库。
*   **专利库优先顺序：** CNTXT、ENTXT(C)、DWPI（用于补充）。
    *   **语言策略：** 对于外文申请，若英文表达统一（如专业术语明确），优先用英文库；若表达多样，优先用中文翻译库。

#### 2. 分类号使用规范 (IPC vs CPC)
*   **IPC的局限性：** 虽能圈定大组，但对“检测原理”和“检测细节”区分不够（例如G01M11包含了所有光学测试，噪音极大）。
*   **CPC的优势与应用：** **强烈推荐**。CPC在G01M下进行了细分，能大幅提高信噪比。
    *   **G01M3/00 (密封性)：** CPC对不同检测手段（示踪气体、压力衰减等）有极细致划分。
    *   **G01M11/00 (光学测试)：** 新增了 **G01M11/30** 等小组，解决了IPC下该组文献过多的问题。

---

### 二、 语义检索策略 (Semantic Retrieval) —— “定点爆破”

语义检索在G01M中主要用于解决分类号不准或跨领域难题。文档强调了三种具体的**基准调整策略**：

#### 1. 以申请号为基准（默认策略）
*   **适用场景：** 申请本身分类号（IPC/CPC）非常准确（如 **G01M13/04 轴承测试**、**G01M3/00 密封测试**），且技术主题单一。
*   **操作：** 直接利用系统默认的申请号基准，利用分类号圈定模型进行筛选。

#### 2. 改写文本作为语义基准（核心策略）
*   **适用场景 A（分类偏差）：** 申请被归类到“应用领域”而非“测试领域”（例如：锂电池密封测试被分入H01M电池制造，而非G01M）。
    *   *对策：* 手动输入一段描述**测试原理**的文本（如“利用无水硫酸铜遇水变色检测电池密封性”）。
*   **适用场景 B (权利要求过宽)：** 权利要求写得非常上位（如“驱动电路时序控制”），实际用于特定场景（如“OIS光学防抖测试”）。
    *   *对策：* 将说明书中的**具体应用场景**（背景技术、有益效果）提取出来，改写成语义文本。
*   **适用场景 C (部件改进)：** 创新点在于测试台的一个常规部件（如“收线机构”），与测试主题无关。
    *   *对策：* 剥离测试背景，专门针对该**部件的结构和功能**改写文本。

#### 3. 调整语义分词/加权（精细化微调）
*   **适用场景：** 自动分词未能捕捉到核心特征，或者分类号圈定导致漏检。
*   **操作：**
    *   **打破分类号限制：** 通过调整分词，可以消除分类号对结果的强行限定。
    *   **加权关键词：** 当核心词（如“防爆”、“种植体”）在默认排序中权重过低（例如仅为1）时，**手动将其权重调高（如调至3）**，强制系统将含有该特征的文献排在前面。

---

### 三、 融合检索策略 (Fusion Retrieval) —— “分门别类”

G01M各小类的技术特点不同，需采用不同的布尔+语义融合策略：

#### 1. 优先使用分类号检索的领域
这些领域的IPC或CPC分类位置准确，噪音低，可直接用 `分类号 AND 关键词`。
*   **G01M3/+ (密封性)：** 分类非常准（如 /02真空，/04漏液，/38光照）。**注意：** 需结合CPC（如G01M3/2815 压力衰减法）以进一步降噪。
*   **G01M13/+ (机械部件)：** 针对特定部件（/04轴承，/02齿轮）分类极准，直接检索。
*   **G01M1/+ (动静平衡)：** 领域集中，细分度高。

#### 2. 需扩展分类号或慎用分类号的领域
*   **G01M17/+ (车辆测试)：**
    *   *难点：* 综合性强，常与 **G01M15 (发动机)** 交叉。
    *   *策略：* 检索时需将分类号扩展到大组，甚至考虑 **G01N3/+ (材料强力测试)**。
*   **G01M11/+ (光学测试)：**
    *   *难点：* 文献量巨大，IPC细分不够。
    *   *策略：* 必须依赖 **CPC** 进行细分，同时关注 **G01J (光测量)** 领域的交叉。

#### 3. 优先使用关键词检索的领域
当分类号无法有效区分或文献量极少时，采用 `关键词构建布尔式`，再用语义排序。
*   **G01M9/+, G01M10/+ (流体/空气动力学)：** 文献少，分类号粗。建议直接用“风洞”、“流场”等关键词。
*   **G01M99/00 (其他/未分类)：** 这是一个“垃圾桶”分类号，包含大量杂项。
    *   *策略：* 必须依靠精准的关键词组合。
*   **复杂机械结构（无特定分类）：** 如“尾气管密封工装”。
    *   *策略：* 使用**临近算符**（如 `10w`, `/s`）来限定部件之间的位置关系（如 `(夹具 10w 管)`），以排除仅包含关键词但结构不相关的噪音。

### 四、 针对性检索场景实操指南

#### 1. 涉及“标准/法规”的申请
*   **特征：** 涉及车灯校核、车辆碰撞安全等合规性测试。
*   **策略：** 必须在非专利库中检索**国家标准（GB）、行业标准**。很多发明构思实质上是现有标准的数字化或自动化实现。

#### 2. 涉及“公式/模型/算法”的申请
*   **特征：** 燃烧噪声计算、轴承故障诊断算法、神经网络。
*   **策略：**
    *   优先查 **CNKI（硕博论文）** 和 **IEEE**。
    *   关键词提取公式中的物理量参数（如“一级影响”、“敏感性系数”）。
    *   不要局限于申请人，直接用技术点在学术库中“大海捞针”。

#### 3. 涉及“交叉领域”的申请
G01M常与其他领域发生交叉，单纯检索G01M会漏检。
*   **G01M + G06T (图像处理)：** 如基于视觉的轴承检测、质心检测。
    *   *策略：* 检索式需包含 `G06T` 分类号或图像处理关键词（二值化、像素、卷积神经网络）。
*   **G01M + 医疗器械 (A61)：** 如手术器械测试、牙种植体测试。
    *   *策略：* 舍弃G01M分类号，直接用“手术钳”、“种植牙”等具体对象关键词跨库检索。
*   **G01M + G01N (材料测试)：** 涉及结构强度的测试（如加压、加载）。
    *   *策略：* 扩展检索 **G01N3/**。

---

### 五、 检索流程图解

1.  **审题：** 确定测试对象、原理、技术手段。
2.  **分类号预判：**
    *   准确吗？ -> 是 (G01M3, 13, 1) -> **策略：申请号基准语义 / 分类号+关键词融合**。
    *   一般？(G01M17, 11) -> **策略：CPC细分 / 扩展到G01N, G01J**。
    *   不准/跑偏？(G01M99, 分入应用类) -> **策略：改写文本语义 / 纯关键词逻辑 / 临近算符**。
3.  **非专利检索判断：**
    *   是高校/科研所？涉及算法/标准？ -> **策略：立刻去CNKI/万方/标准库**。
4.  **调整与迭代：**
    *   噪音大？ -> 调整语义分词权重 / 使用CPC / 增加临近算符。
    *   漏检？ -> 去掉分类号 / 扩展交叉领域关键词。

## 总结
G01M领域的检索核心在于**灵活多变**。不能死守IPC分类号，必须善用**CPC分类号**进行细化，利用**语义分词调整**和**文本改写**来纠正分类偏差，并且对于涉及算法、模型和标准的申请，要果断转向**非专利数据库**。
