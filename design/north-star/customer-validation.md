# 客户验证 · 审查员深度访谈脚本（议题 G）

**目的**：在投入更多产品/工程资源之前，用 5–8 次深度访谈验证北极星的核心假设。不验证就投入，等同于把整盘棋押在我们对审查员的想象上。

**时长**：每次 60–90 分钟。一对一为主。
**形式**：远程视频（录屏 + 录音，征得同意）。
**输出**：每次访谈一份逐字摘要 + 一张「假设记分卡」。

---

## 1. 要验证的 7 个核心假设

每个假设都必须有**可被证伪的**操作化判据，否则就是同温层自嗨。

| # | 假设 | 操作化判据（被证伪的样子）|
|---|------|------------------------|
| H1 | 审查员当前工作流存在显著效率痛点，且 AI 能帮上忙 | 多数受访者描述的痛点其实是「制度问题」（KPI、合规、组织），AI 解决不了 |
| H2 | 审查员愿意把"做审查"的角色让渡给"审 AI 的审查" | 受访者明确表示这会动摇职业认同 / 担心被替代 / 拒绝监督 AI 而非自己做 |
| H3 | Disagreement-first 比 Copilot 更受欢迎 | 多数受访者偏好"AI 帮我但我自己做"，不愿"AI 做我审" |
| H4 | 审查员能接受 AI 的判决基础上签字（哪怕推理链可审计） | 受访者明确表示无论 AI 多透明，他们都不会在没有亲自重做一遍的前提下签字 |
| H5 | Policy Console 是有价值的（审查员愿意"教"AI）| 受访者认为这是额外工作量，没有动机去维护政策 |
| H6 | Ambient Insight（自审视）是受欢迎的功能 | 受访者明确不希望被展示"你与同处的对比"，认为是监视/打分 |
| H7 | 体制/合规层面允许产品在专利局落地 | 受访者（尤其领导级）告诉我们这种程度的 AI 自动化在政策上不可能批 |

**重要**：访谈中**不要**给受访者看到这些假设。我们要去测他们「自然」的反应。

---

## 2. 受访者画像（目标样本）

最少 5 人、目标 8 人。多样性比数量重要。

| 维度 | 目标分布 |
|------|---------|
| **资历** | 资深审查员（10+ 年）至少 2 位；中段（5–10 年）2–3 位；新人（2–5 年）1–2 位 |
| **机构** | CNIPA 北京 / 苏州 / 河南审协中心 至少各 1 位；USPTO 或 EPO（如能接触）至少 1 位 |
| **领域** | 电学、机械、化药、生物至少各 1 位；其中含 AI/软件类至少 2 位 |
| **角色** | 一线审查员为主；尝试接触 1 位审查主管 / 处长（测体制约束）|
| **状态** | 含 1–2 位**前**审查员（已离职到代理所或企业），可以更坦白讲体制内不能说的话 |

---

## 3. 招募渠道

按性价比排序：

1. **个人/团队人脉直接介绍** — 最高信任度，对方愿意讲真话
2. **代理所合作伙伴介绍** — 代理所天然认识审查员
3. **知识产权学院校友** — 中国政法、人民大学、华东政法、UNH 等
4. **LinkedIn 冷启** — 找前 USPTO/EPO examiner，更愿讲
5. **行业会议旁聊** — 中国知识产权年会、AIPLA、INTA 等
6. **付费招募平台** — Respondent.io、用户邦等（成本高、质量参差）

**激励**：CNIPA 在职审查员不收酬金（合规风险）→ 用"早鸟体验权 + 你的反馈会出现在产品里"作为价值。前审查员/代理人可付费 $200–500 / 时。

**预约话术（模板）**：
> "您好，我在做一个面向专利审查员的 AI 工具研究，希望听听您实际审查工作中的痛点和对 AI 的看法。这不是销售推介，纯粹学习。1 小时左右，不录视频可只录音，所有内容会做匿名化处理。"

---

## 4. 访谈结构（time-boxed, 90 min）

| 阶段 | 时长 | 目的 |
|------|------|------|
| ① 开场 + 背景 | 5 min | 建立信任，了解受访者基本盘 |
| ② 当前工作流 deep dive | 20 min | **在不暴露我们的方案前**深挖痛点 |
| ③ 痛点排序 + 偏好 | 10 min | 让受访者自己排，避免被我们的框架带偏 |
| ④ 概念性介绍北极星 | 5 min | **不展示原型**，只用一段话描述 |
| ⑤ 原型 walk-through + 反应 | 30 min | 给四个视图分别记录第一反应 |
| ⑥ 体制 / 合规 / 阻碍 | 10 min | 测 H7 |
| ⑦ 收尾 + 引荐 | 10 min | 拿到下一位受访者 |

---

## 5. 提问列表

### ① 开场（5 min）
- 您在 [机构] 工作多久了？主要审什么领域？
- 一天大概审多少件？多少时间在检索、多少在写 OA、多少在内部协调？
- 您对"AI"在专利审查中的现状有什么印象？（**听他们用什么词**：辅助 / 替代 / 危险 / 提效 / 不靠谱）

### ② 当前工作流（20 min）— 测 H1
> 关键原则：不诱导，只问开放问题，让他们讲。

- 能不能从您今天最近审的一件案子开始，告诉我您是怎么处理的？（从案件分配到下发文书）
- 这个过程中哪一步最让您头疼？为什么？
- 哪一步最枯燥但必须做？（这往往是 AI 最容易接管的）
- 您有没有觉得"这一步根本不该是我做"的环节？
- 如果给您多一倍时间，您会用来做什么？
- 您审过的案子里，**返工率**多大？什么原因导致返工？
- 同事/主管复核您的审查，他们最常 challenge 哪一类判断？

**听什么**：
- 痛点是否清晰、具体（不是"工作好累"，而是"检索一篇专利平均要切换 3 个数据库"）
- 受访者对效率提升的**真实**态度（不是"当然好啊"，而是"那我下班可以早一点吗 / 反正 KPI 不会变"）
- "制度性痛点" vs "可被 AI 解决的痛点"的边界

### ③ 痛点排序 + AI 期待（10 min）
- 我刚才听您讲了 X / Y / Z 这几个痛点，您给它们排个优先级好不好？
- 这几个痛点里，您觉得**哪些能被 AI 解决，哪些不能**？为什么？
- 您理想中的"AI 帮你审"是什么样的？请具体一些。
- 在这个理想形态里，您自己做什么、AI 做什么？

**听什么**：自然语言里他们是说"AI 帮我"还是"AI 替我"？这直接关系 H2/H3。

### ④ 概念介绍（5 min）
> 用一段话描述，**不展示原型**：
>
> "我们在思考一种产品：AI 会自动跑完案件的全流程——检索、对比、创造性判断、起草 OA。审查员的工作变成审阅 AI 的判断、在分歧点裁决、为关键案件签字。AI 学习您的判决标准，越用越像您。您怎么看？"

- 您的第一反应是什么？
- 这个概念有什么让您觉得**会卡壳**的地方？
- 您愿意/不愿意用这样一个产品的最大原因分别是什么？

**听什么**：
- 直觉性的拒绝信号（H2 风险）
- 「这在我们单位不可能批」（H7 风险）
- 「我宁可自己做」vs「这正好是我想要的」

### ⑤ 原型 walk-through（30 min）
> 顺序：Decision Queue → Reasoning Card → Artifact Canvas → Policy Console → Ambient Insight。每个视图 5 min。
>
> **原则**：每展示一个视图，先沉默 10 秒，让受访者自己说。**不要解释**，等他们提问再回答。

对每个视图都问的三件事：
1. 您看到的第一眼，您觉得这是干嘛的？（测可读性）
2. 如果您每天上班打开看到这个，您会做什么？（测意图明确性）
3. 这里**最让您不舒服**的是什么？最让您兴奋的是什么？

**特定视图问题**：
- **Decision Queue**：如果我说今天给您 5 件需要您裁决，但 AI 帮您处理了 42 件您没看过的——您能睡得着吗？您会真的去抽样审计吗？
- **Reasoning Card**：AI 给您写了一份判决书，您看了一眼说"对"就签字——这件事您内心能接受吗？您会不会还是想自己重做？
- **Artifact Canvas**：现在大部分时间您在这里改 OA。AI 写的 OA 您会怎么改？改完会不会觉得"那我自己写更快"？
- **Policy Console**：您愿意花时间维护这个东西吗？如果不维护，AI 就会跟您不一致——这值得吗？
- **Ambient Insight**：这里有一项"你与同处审查员的对比"——这让您觉得是**有用反馈**还是**被监视**？

### ⑥ 体制 / 合规 / 阻碍（10 min）— 测 H7
- 您觉得在 [机构] 部署这样一个产品，最大的障碍是什么？
- 您的主管 / 上级 / 党组会怎么看？
- 如果今天就让您用，您单位会有人 block 这件事吗？谁？为什么？
- 如果 AI 判错了案子被复审推翻，谁的责任？这会影响您用它吗？
- 工会 / 同事会怎么反应？

### ⑦ 收尾（10 min）
- 我们今天聊的内容，您觉得我**没问对/没问到**的关键问题是什么？
- 您能介绍 1–2 位您觉得我应该再聊聊的同行吗？
- 我们的产品做到什么程度，您愿意第一批试用？

## 6. 听什么、不听什么

| 信号 | 含义 |
|------|------|
| ✅ **具体故事 + 数字** | "上周一件案子我光检索就花了 4 小时" — 真痛点 |
| ✅ **皱眉 + 长沉默** | 触到了体制性问题 / 不能说的话，重点追问 |
| ✅ **主动追问产品细节** | 真感兴趣，往里挖偏好 |
| ✅ **「这个我以前想过」** | 他们的同类需求强烈 |
| ⚠️ **「这个挺好的」（无具体）** | 礼貌客套，**不算正面信号** |
| ⚠️ **「我觉得对其他人有用」** | 自己不会用，重新打开追问 |
| ⚠️ **「肯定会用 / 100% 想要」** | 警惕过度热情，引导讲具体使用场景 |
| 🚩 **反复回到"返工 / 责任 / 不能错"** | 法律责任是核心顾虑，H4 风险 |
| 🚩 **「我们单位不可能」** | H7 风险，必须深入 |

**特别警惕**：审查员（尤其在职的）有很强的「政治正确」表达习惯，倾向于说"AI 是辅助、人不能被替代"这种官方话术。不要被这些表态误导，**追问具体场景**。

## 7. 红旗信号（看到立刻调整北极星）

| 红旗 | 触发条件 | 应对 |
|------|---------|------|
| **A · 角色拒绝** | 5+ 人中有 3+ 明确表示不愿意从"做审查"转为"审 AI" | **H2/H3 被证伪** → 退回 Copilot 形态 |
| **B · 签字恐惧** | 5+ 人中有 4+ 表示哪怕推理链完整也不会盲签 | **H4 被证伪** → MVP 必须把"重做一遍的轻量版"作为主路径 |
| **C · 体制 hard no** | 任何一位受访的领导级明确说"这种程度专利局不会批" | **H7 严重风险** → 重新评估 GTM；可能要先做 attorney 侧 |
| **D · Ambient 反弹** | 5+ 人中有 3+ 表示自审视/同处对比让他们不舒服 | **H6 调整** → 这个视图设计为可关闭、个人私有 |
| **E · Policy 拒绝** | 5+ 人中有 4+ 表示不会花时间维护政策 | **H5 调整** → 政策必须从行为隐式提炼，不能要求显式维护 |

## 8. 输出格式

每次访谈结束 24 小时内填一张**假设记分卡**：

```
受访者 #N · [初步匿名 ID] · [日期]
背景：[资历 / 机构 / 领域]
痛点 top 3：
  1. ...
  2. ...
  3. ...

假设记分（−2 强证伪 / −1 弱证伪 / 0 中性 / +1 弱支持 / +2 强支持）：
  H1 痛点存在     [  ]  关键引述："..."
  H2 角色让渡     [  ]  关键引述："..."
  H3 Disagree>Copilot  [  ]
  H4 接受签字     [  ]
  H5 政策可维护    [  ]
  H6 Ambient 受欢迎  [  ]
  H7 体制可批     [  ]

惊喜（没预料到的）：
  ...

让我们重新思考的：
  ...
```

5–8 份记分卡合成总分。**任何一个假设总分低于 −3，触发对应的红旗动作**（见上表）。

## 9. 时间线建议

| 周 | 动作 |
|---|------|
| W1 | 招募 + 试访谈 1 人（前审查员，安全练手）|
| W2 | 访谈 #2 – #4 |
| W3 | 中期 review，根据假设记分调整后续访谈重点 |
| W4 | 访谈 #5 – #8 |
| W5 | 合成结论，更新北极星 + 输入到 MVP 切片设计 |

**总时间 5 周。如果 W2/W3 已出现明显红旗，立刻暂停后续访谈，先调整方向。**

---

## 10. 给访谈者本人的纪律

- **不卖产品**，只学
- **不修复**，受访者讲"这里有问题"时不要急着说"我们会解决"
- **追问 3 次**：每个有意思的回答后追问"为什么" / "举个例子" / "上次发生是什么时候"
- **承认无知**：你不是审查员，多说"我对这块不熟悉，能多讲讲吗"
- **观察沉默**：3 秒以上的沉默通常意味着真实想法在浮现，不要打断
